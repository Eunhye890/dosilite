# 공유오피스 일정 관리 시스템 기획서

> 버전: v1.1 | 작성일: 2026.03.10 | 변경: 모바일 최적화 MVP 포함

---

## 1. 프로젝트 개요

### 목적
관리자 2명이 공유오피스 예약 현황을 실시간으로 함께 관리하기 위한 내부용 웹 시스템

### 핵심 원칙
- **심플하게**: 관리자 2명만 사용, 불필요한 기능 없음
- **빠르게 만들기**: 클로드 코드로 빌드, 최소 구성
- **공유 편하게**: URL 하나로 접근, 로그인 불필요

---

## 2. 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | Next.js (App Router) | 클로드 코드 친화적, 빠른 개발 |
| 데이터베이스 | Supabase (PostgreSQL) | 무료, 실시간 동기화, 설정 간단 |
| 배포 | Vercel | 무료, GitHub 연동 자동 배포 |
| 스타일 | Tailwind CSS | 빠른 UI 구성 |
| 달력 컴포넌트 | react-calendar 또는 직접 구현 | |

### 아키텍처 흐름
```
브라우저 (관리자 A/B)
    ↓ URL 접속
Next.js (Vercel 배포)
    ↓ API Route
Supabase (DB + 실시간)
```

---

## 3. 접근 방식 (인증)

- **로그인 없음**: URL 자체가 접근 키
- 배포 URL을 관리자 2명끼리만 공유
- 보안이 더 필요해질 경우 추후 간단한 비밀번호 레이어 추가 가능

---

## 4. 데이터 모델

### reservations 테이블

```sql
CREATE TABLE reservations (
  id          UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  date        DATE NOT NULL,
  start_time  TIME NOT NULL,
  end_time    TIME NOT NULL,
  title       TEXT NOT NULL,        -- 예약 제목 (고객명 or 용도)
  memo        TEXT,                 -- 메모/특이사항 (선택)
  created_by  TEXT,                 -- 등록한 관리자 이름
  created_at  TIMESTAMPTZ DEFAULT now(),
  updated_at  TIMESTAMPTZ DEFAULT now()
);
```

## 5. 화면 구성

### 5-1. 메인 페이지 (`/`)

**달력 영역**
- 월 단위 달력 표시
- 예약이 있는 날짜에 시각적 표시 (점 또는 색상)
- 이전/다음 달 이동 버튼

**예약 현황 영역** (달력 하단)
- 선택한 날짜의 예약 목록
- 기본: 오늘 날짜 선택된 상태로 표시
- 각 예약 카드에 표시: 시간 / 제목 / 메모 / 등록자
- 수정 버튼 / 삭제 버튼 (각 카드에)

```
┌─────────────────────────────┐
│   < 2026.03 >               │
│   일 월 화 수 목 금 토       │
│   ...  달력 그리드  ...      │
│   (예약 있는 날 ● 표시)     │
└─────────────────────────────┘
┌─────────────────────────────┐
│  3월 10일 예약 현황          │
│  ┌───────────────────────┐  │
│  │ 13:00~15:00           │  │
│  │ 홍길동 고객            │  │
│  │ 메모: 회의실 세팅 요망  │  │
│  │              [수정][삭제] │
│  └───────────────────────┘  │
│  [+ 예약 추가]              │
└─────────────────────────────┘
```

### 5-2. 예약 등록/수정 모달

- 달력에서 날짜 클릭 → 모달 오픈
- 또는 [+ 예약 추가] 버튼 클릭

**입력 필드**
| 필드 | 타입 | 필수 |
|------|------|------|
| 날짜 | date picker | ✅ |
| 시작 시간 | time picker | ✅ |
| 종료 시간 | time picker | ✅ |
| 예약 제목 | text input | ✅ |
| 메모/특이사항 | textarea | ❌ |
| 등록자 | text input | ❌ |

## 6-A. 모바일 최적화 상세

관리자가 현장에서 폰으로 빠르게 확인/등록할 수 있도록 모바일 퍼스트로 설계

### 레이아웃 원칙
- **기준 너비**: 375px (iPhone SE 기준) 이상 모두 정상 동작
- **터치 타겟**: 모든 버튼/날짜 셀 최소 44×44px 이상
- **폰트**: 본문 16px 이상 (모바일 자동 확대 방지)
- **스크롤**: 달력 → 예약 목록 순서로 자연스럽게 위→아래 스크롤

### 달력 모바일 처리
- 날짜 셀이 손가락으로 탭하기 충분한 크기로 렌더링
- 예약 있는 날짜: 작은 컬러 점(●) 표시 (텍스트 아래)
- 이전/다음 달 버튼: 충분히 큰 탭 영역

### 예약 등록 모달 → 바텀 시트로 변경
- 데스크탑: 중앙 모달
- 모바일: **바텀 시트** (화면 아래에서 올라오는 방식) — 엄지 손가락으로 조작 편함
- 입력 필드는 세로 한 줄씩 배치, 넓은 탭 영역
- 시간 입력은 네이티브 `<input type="time">` 사용 (모바일 OS 기본 피커 활용)

### 예약 카드 모바일 처리
- 카드 전체가 터치 가능
- 수정/삭제는 카드 내 아이콘 버튼 (텍스트 대신 아이콘으로 공간 절약)
- 스와이프 삭제는 v2로 미룸 (구현 복잡도 대비 우선순위 낮음)

### 하단 고정 버튼
- [+ 예약 추가] 버튼을 화면 우측 하단에 **FAB(Floating Action Button)** 으로 고정
- 스크롤해도 항상 접근 가능

```
모바일 화면 구조
┌──────────────────┐
│  < 2026.03 >     │
│  일 월 화 수 목 금 토 │
│  [달력 그리드]   │
│  (예약 있는 날 ●) │
├──────────────────┤
│  3월 10일 예약   │
│  ┌────────────┐  │
│  │ 13:00~15:00│  │
│  │ 홍길동     │  │
│  │ 메모 내용  │  │
│  │    ✏️  🗑️ │  │
│  └────────────┘  │
│                  │
│              [+] │  ← FAB 고정
└──────────────────┘

바텀 시트 (예약 추가/수정 시)
┌──────────────────┐
│  (배경 딤처리)   │
├──────────────────┤
│  ▬ (드래그 핸들) │
│  예약 추가       │
│  날짜: [____]    │
│  시작: [__:__]   │
│  종료: [__:__]   │
│  제목: [______]  │
│  메모: [______]  │
│  [취소] [저장]   │
└──────────────────┘
```

### Tailwind 모바일 구현 포인트
```
// 달력 날짜 셀
className="min-h-[44px] min-w-[44px] touch-manipulation"

// FAB 버튼
className="fixed bottom-6 right-6 w-14 h-14 rounded-full shadow-lg"

// 바텀 시트
className="fixed bottom-0 left-0 right-0 rounded-t-2xl bg-white p-6"

// 시간 입력 (모바일 네이티브 피커)
<input type="time" className="w-full text-base p-3 border rounded-lg" />
```

## 7. 클로드 코드 빌드 순서

Claude Code에서 아래 순서로 개발 진행 권장:

```
Step 1. 프로젝트 초기화
  - npx create-next-app@latest
  - Supabase 프로젝트 생성 및 환경변수 설정

Step 2. DB 설정
  - Supabase에서 reservations 테이블 생성
  - RLS(Row Level Security) 비활성화 (내부 툴이므로)

Step 3. 달력 UI 구현 (모바일 퍼스트)
  - 월 달력 컴포넌트 (터치 타겟 44px 이상)
  - 날짜 선택 상태 관리
  - 예약 있는 날짜 점 표시

Step 4. 예약 CRUD
  - 예약 목록 조회 API
  - 예약 추가/수정/삭제 API
  - 바텀 시트 컴포넌트 (모바일) / 모달 (데스크탑)

Step 5. 모바일 UX 마무리
  - FAB 버튼 고정
  - 네이티브 time picker 적용
  - 반응형 레이아웃 점검 (375px ~ 1280px)

Step 6. 실시간 동기화
  - Supabase Realtime 구독 설정
  - 다른 관리자가 추가하면 자동 반영

Step 7. Vercel 배포
  - GitHub에 push
  - Vercel 연동 및 환경변수 설정
```
